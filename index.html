Version control with Git

This course is based on materials by Software Carpentry

Prerequisites

In this lesson we use Git from the Unix Shell. Some previous experience with the shell is expected, but is not mandatory.

You will need a GitHub account for the afternoon session. Please sign up for an account this morning if you do not already have an account.

Resources

Workshop presentation

Git cheat sheet (You may want to download a copy of this for reference during this workshop)

Topics

  1. Introduction - Why use version control?
  2. Setting up and working with a local repository
  3. Looking at history and differences
  4. Best practice for committing changes
  5. Branching, merging and resolving conflicts
  6. Undoing changes
  7. Rebasing
  8. Using remote repositories to work from different locations
  9. Collaborating using a remote repository
  10. Pull requests
  11. Hints and tips
  12. Conclusion
